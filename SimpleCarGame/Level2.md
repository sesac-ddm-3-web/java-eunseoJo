# 기능 요구사항 정의 
- 주어진 횟수 동안 n대의 자동차는 전진 OR 정지
- 사용자는 (몇대의 자동차, 몇번의 이동) 입력
- 전진하는 조건 : 0-9사이의 무작위 값 >= 
- 자동차의 상태를 화면에 출력
  - 한번의 이동이 끝날때마다 출력 
- 모든 로직에 단위테스트 구현
  - UI로직 제외
- UI로직은 racing.view.InputView, ResultView 같은 클래스로 추가해 분리

# 클래스 및 자료구조 
## Car 
- 차의 현재 위치를 저장
- 차를 이동하는 메서드,  현재 위치를 가져오는 메서드를 관리

## InputView
- 입력을 담당한다.
- 몇대의자동차, 몇번의 이동을 입력받을지에 대한 메서드 

## OutView 
- 출력을 담당한다. 


## RaceController
- 게임진행에 필요한 메서드 정의 
  - 모든 차를 움직이는 메서드
  - 각 턴을 진행하는 메서드
  - 게임을 실행하는 runGame메서드

## NumberGenerator, CarGenerator 
- 테스트를 위해서 인터페이스로 구현 
- 랜덤으로 값을 생성하는 메서드
- 차를 생성하는 메서드

# 테스트
- 자동차가 4이상이면 전진하는지를 테스트한다.
- 자동차가 4미만일때 정지하는지를 테스트한다.
- 자동차 생성이 잘 이루어지는지를 테스트한다. 
- 모든 자동차가 잘 이동하는지를 테스트한다.
